# MarkPin 개인정보처리방침

시행일: 2026년 3월 30일 | 최종 수정: 2026년 3월 30일

MarkPin(이하 "확장프로그램")은 사용자의 개인정보를 소중히 여기며, 관련 법령에 따라 아래와 같이 개인정보를 처리합니다.

---

## 1. 수집하는 정보

MarkPin은 다음 정보를 수집하거나 저장합니다:

- **북마크 데이터**: 사용자가 저장한 웹페이지의 URL, 제목, 설명, 카테고리, 태그
- **API 인증 토큰**: 사용자가 직접 입력한 GitHub Personal Access Token, Notion Integration Token (선택 사항)
- **확장프로그램 설정**: 동기화 간격, 분류 모드, 언어 설정 등 사용자 환경설정

> MarkPin은 사용자의 이름, 이메일, 위치 정보 등 개인 식별 정보를 수집하지 않습니다.

---

## 2. 정보의 저장 위치

- **로컬 저장소**: 북마크 데이터는 `chrome.storage.local`에 브라우저 내부에만 저장됩니다.
- **동기화 저장소**: 확장프로그램 설정과 API 토큰은 `chrome.storage.sync`에 저장되며, Chrome 계정에 연동된 기기 간 동기화됩니다.

---

## 3. 외부 서비스 연동

사용자가 **선택적으로** 활성화한 경우에만 다음 외부 서비스로 데이터가 전송됩니다:

- **GitHub API** (`api.github.com`): 북마크를 Markdown 파일로 백업. 사용자의 GitHub 저장소에만 저장됩니다.
- **Notion API** (`api.notion.com`): 북마크를 Notion 데이터베이스에 동기화. 사용자의 Notion 워크스페이스에만 저장됩니다.

> MarkPin은 자체 서버를 운영하지 않으며, 수집된 데이터를 제3자에게 판매하거나 공유하지 않습니다.

---

## 4. API 토큰 보안

사용자가 입력한 GitHub/Notion 토큰은 Chrome 브라우저의 `chrome.storage.sync`에 저장됩니다. 이 저장소는 Chrome 계정으로 보호되며, 해당 확장프로그램만 접근할 수 있습니다. 토큰은 외부 서버로 전송되지 않으며, 해당 API 호출 시에만 사용됩니다.

---

## 5. 데이터 삭제

사용자는 언제든지 다음 방법으로 데이터를 삭제할 수 있습니다:

- 확장프로그램 설정에서 API 토큰 제거
- Chrome 확장프로그램 관리에서 MarkPin 제거 (모든 로컬 데이터 삭제)
- Chrome 설정에서 확장프로그램 데이터 초기화

---

## 6. 권한 사용 목적

| 권한  | 사용 목적 |
| --- | --- |
| `activeTab` | 현재 탭의 URL과 제목을 읽어 북마크로 저장 |
| `storage` | 북마크와 설정을 브라우저에 저장 |
| `alarms` | 주기적인 동기화 스케줄 실행 |
| `sidePanel` | 사이드 패널에서 북마크 목록 표시 |

---

## 7. 변경 사항

본 개인정보처리방침이 변경될 경우, 확장프로그램 업데이트 노트를 통해 안내합니다.
